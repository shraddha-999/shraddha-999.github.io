RateGuard — Distributed Rate Limiter

Production-grade HTTP rate limiter with three algorithm implementations, atomic Redis Lua scripts, Prometheus observability, and a full React 19 SaaS dashboard with real-time charts and a request simulator.

⚠ The Challenge

Concurrent rate-limit checks create race conditions where multiple in-flight requests can all observe "quota available" before any write commits — silently breaking the limit guarantee under load.

✓ The Solution

All three algorithms (Fixed Window, Sliding Window Log, Token Bucket) use atomic Lua scripts executed via Redis EVALSHA — the entire read-modify-write cycle is a single Redis operation. 49 tests pass including thread-safety and concurrency stress tests with zero race conditions observed.

Python · FastAPI Redis · Lua React 19 · TypeScript Prometheus · Grafana Docker Compose
Live Demo Source Code
Architecture highlights
⚡

3 algorithms — Fixed Window (HASH+TTL), Sliding Window (ZSET log), Token Bucket (HMGET/HMSET) — each switchable per API config with hot-reload
